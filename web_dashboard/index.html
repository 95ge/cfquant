使用说明

              LTtx / QMT / Web
            

            

              

                
                
                
                
                
                
                
                
                
                
              


              

                

                  

                    
项目介绍

                    
cfquant 是一个运行在本机的 QMT 交易桥接控制台。它把网页控制台、外部 Python 程序和 QMT 策略脚本连接起来，让外部程序可以通过统一接口完成资金查询、持仓查询、委托查询、下单、撤单和实时回调接收。

                    
系统里最重要的概念是桥接端和账号绑定。桥接端表示一套 QMT 入口，通常对应一个正在运行的 QMT 客户端；账号绑定表示某个资金账号应该走哪个桥接端。这样在多账号、多 QMT 同时运行时，请求仍然可以被准确转发到正确终端。

                  


                  

                    
架构图

                    

                      

                        网页控制台
                        资产、持仓、委托、交易、绑定、接口调试
                      

                      

                        外部程序
                        Python、量化服务、风控服务、自动化工具
                      

                      
HTTP / WebSocket

                      

                        cfquant Web Server
                        API 鉴权、账号路由、请求编排、回调缓存
                      

                      
LTtx 本地消息通道

                      

                        LTtx 通信层
                        连接管理、消息转发、桥接端状态维护
                      

                      
bridge_id / account_id

                      

                        QMT 策略脚本
                        CFQUANT.py 查询桥，CFQUANT_TRADE_LOWLAT.py 极速交易桥
                      

                      
QMT 原生接口

                      

                        QMT 交易终端
                        登录账号、柜台连接、实际查询与交易权限
                      

                    

                    

                      请求路径
                      外部请求先到 Web Server，再按账号绑定找到桥接端，通过 LTtx 发给 QMT 脚本，最后由 QMT 终端执行。委托、成交、资金、持仓回调按相反方向返回到网页和 WebSocket 客户端。
                    

                  


                  

                    
核心文件

                    

                      

                        
                          

                            文件或模块
                            作用
                          

                        
                        
                          

                            cfquant_web_server.py
                            提供 Web 控制台、HTTP API、WebSocket 回调和配置保存能力。
                          

                          

                            web_dashboard
                            浏览器端页面，负责交易操作、接口调试、状态查看和教程展示。

                            启动本地服务和通信组件的入口。
                          

                        
                      

                    

                  

                


                

                  

                    
快速开始

                    
README 只负责帮助你启动本地 Web 控制台。真正接入 QMT 时，需要在本页继续完成部署、桥接端配置、账号绑定和验证。

                    
    
                      
  1. 启动 Web：双击 cfquant\start_cfquant.bat，打开 http://127.0.0.1:8765/。
    2. 
                      
  2. 部署到 QMT：按左侧“部署到 QMT”章节，把依赖目录和桥接脚本放到 QMT 的 Python 策略目录。
    3. 
                      
  3. 加载桥接脚本：在 QMT 中加载 CFQUANT.py 和 CFQUANT_TRADE_LOWLAT.py。
    4. 
                      
  4. 配置桥接端：单 QMT 使用 default；多 QMT 按“桥接端配置”章节设置不同 USER_BRIDGE_ID。
    5. 
                      
  5. 绑定账号：进入“绑定”页面，把资金账号绑定到对应桥接端。
    6. 
                      
  6. 验证连接：刷新状态和资产持仓；普通和极速通道都在线后，再接入外部程序或进行交易操作。
    7. 
                    

                  


                  

                    
启动后先看哪里

                    

                      概览
                      确认资产、持仓、实时委托和绑定实时状态是否有数据。
                      状态
                      查看 LTtx、普通 QMT、极速交易端是否在线，以及最近一次检测时间。
                      绑定
                      维护桥接端和账号关系，适合排查账号请求发错 QMT 的问题。
                      接口
                      给外部程序接入前，先在这里调试 HTTP 和 WebSocket 参数。
                    

                  

                


                

                  

                    
Python 接入前提

                    
外部 Python 程序可以把 cfquant 当作 xtquant 兼容层使用。使用前需要先启动本地 start_cfquant.bat，并在 QMT 中加载 CFQUANT.py 和 CFQUANT_TRADE_LOWLAT.py。Web 控制台里也要先完成资金账号和桥接端绑定。

                    

                      cfquant 不是直接连接柜台的独立交易客户端
                      外部 Python 请求会先连接本机 LTtx，再由 QMT 里的桥接脚本调用 QMT 原生能力。因此 QMT 必须登录账号并保持桥接脚本运行。
                    

                  


                  

                    
安装方式

                    
cfquant 暂未发布到 PyPI，因此暂不支持 pip install cfquant。当前只支持从本地源码目录安装。

                    
cd D:\cfquant
pip install -e .

# 或安装当前源码版本
pip install .

                    
如果不想使用 pip，也可以把源码里的 cfquant\ 和 LTtx\ 两个目录复制到你的 Python 项目根目录，或复制到当前 Python 环境的 site-packages 目录。

                  


                  

                    
替代 xtquant 导入

                    
原来使用原生 xtquant 的代码：

                    
from xtquant import xtdata
from xtquant.xttrader import XtQuantTrader
from xtquant.xttype import StockAccount

                    
可以改成：

                    
from cfquant import xtdata
from cfquant.xttrader import XtQuantTrader
from cfquant.xttype import StockAccount

                    
如果原项目里大量使用 import xtquant，可以用别名方式减少改动：

                    
import cfquant as xtquant

data = xtquant.xtdata.get_full_tick(["000001.SZ"])

                  


                  

                    
连接配置

                    
默认连接本机 LTtx。单 QMT 场景通常不需要手动调用 configure；只要 start_cfquant.bat 已启动、QMT 侧桥接脚本已加载，xtdata 可以像原生 xtquant.xtdata 一样直接使用。

                    
只有需要修改 LTtx 地址、端口、token、超时时间，或给没有账号映射的请求指定默认桥接端时，才需要调用 configure：

                    
from cfquant import configure

configure(
    host="127.0.0.1",
    port=2049,
    token="LTtx",
    timeout=15,
)

                    
也可以通过环境变量配置：

                    
CFQUANT_LTTX_HOST=127.0.0.1
CFQUANT_LTTX_PORT=2049
CFQUANT_LTTX_TOKEN=LTtx
CFQUANT_TIMEOUT=15

                  


                  

                    
行情与交易查询示例

                    
from cfquant import xtdata

tick = xtdata.get_full_tick(["000001.SZ", "600000.SH"])
bars = xtdata.get_market_data(
    field_list=["open", "high", "low", "close", "volume"],
    stock_list=["000001.SZ"],
    period="1d",
    count=5,
)

                    
from cfquant.xttrader import XtQuantTrader
from cfquant.xttype import StockAccount

account = StockAccount("2220009880")
trader = XtQuantTrader("", 0, account=account)
trader.start()

asset = trader.query_stock_asset(account)
positions = trader.query_stock_positions(account)

                    
交易侧按资金账号路由：外部 Python 会读取当前项目目录或 CFQUANT_WEB_CONFIG_FILE 指向的 cfquant_web_config.json，如果里面已有 Web 控制台保存的“资金账号 -> 桥接端”绑定，只需要传同一个资金账号；如果不使用这份 Web 配置，也可以显式指定桥接端：

                    
account = StockAccount("2220009880", bridge_id="qmt2")

                  

                


                

                  

                    
部署到 QMT

                    
QMT 策略脚本运行时，只能稳定导入它所在 Python 策略目录下的文件。部署 cfquant 时，需要把桥接脚本依赖的包和通信文件放到 QMT 能导入的位置。

                    
发布包的外层 cfquant 目录用于启动 Web；其中 cfquant\cfquant\ 是要复制到 QMT 的依赖包，cfquant\qmt_scripts\ 是要放进 QMT 并加载运行的入口脚本。

                    
实际部署时，请找到目标 QMT 安装目录下的 python 文件夹，里面通常会放置各类 .py 策略文件。常见示例是 D:\兴业证券SMT-Q\python、D:\国金证券QMT交易端\python。

                  


                  

                    
需要放到 QMT Python 目录的文件

                    

                      

                        
                          

                            项目文件
                            放置位置
                            用途
                          

                        
                        
                          

                            cfquant\cfquant\ 目录
                            复制到 QMT 的 python\cfquant\
                            桥接核心包，包含普通桥、极速交易桥、频道规则和协议解析。
                          

                          

                            cfquant\qmt_scripts\CFQUANT.py
                            复制到 QMT 的 python\ 根目录
                            普通 QMT 桥入口，负责资金、持仓、委托、成交和状态查询。
                          

                          

                            cfquant\qmt_scripts\CFQUANT_TRADE_LOWLAT.py
                            复制到 QMT 的 python\ 根目录
                            极速交易桥入口，负责低延迟下单、撤单和交易回调。
                          

                          

                            cfquant\qmt_scripts\tx.py
                            复制到 QMT 的 python\ 根目录
                            LTtx 客户端入口，桥接脚本通过它连接本机 LTtx 服务。
                          

                        
                      

                    

                    

                      保持目录层级不变
                      复制后应能看到 python\cfquant\__init__.py、python\CFQUANT.py、python\CFQUANT_TRADE_LOWLAT.py 和 python\tx.py。如果 QMT 提示无法导入 cfquant 或 tx，优先检查文件是否放错目录。
                    

                    

                      目标目录示例
                      如果 QMT 安装在 D:\兴业证券SMT-Q，目标目录通常是 D:\兴业证券SMT-Q\python；如果安装在 D:\国金证券QMT交易端，目标目录通常是 D:\国金证券QMT交易端\python。
                    

                  


                  

                    
部署步骤

                    
    
                      
  1. 关闭或停止旧桥接脚本：如果 QMT 里已经加载过旧版 CFQUANT.py 或 CFQUANT_TRADE_LOWLAT.py，先停止旧策略，避免多个旧实例抢同一频道。
    2. 
                      
  2. 复制依赖包：把发布包里的 cfquant\cfquant\ 整个目录复制到目标 QMT 的 python\cfquant\。
    3. 
                      
  3. 复制入口脚本：把 cfquant\qmt_scripts\ 里的 CFQUANT.py、CFQUANT_TRADE_LOWLAT.py 和 tx.py 放到同一个 QMT python 根目录。
    4. 
                      
  4. 确认本机服务已启动：在发布包外层 cfquant 目录运行 start_cfquant.bat，确保 LTtx 和 Web 控制台已经启动。
    5. 
                      
  5. 在 QMT 加载普通桥：加载 CFQUANT.py，用于查询和普通回调。
    6. 
                      
  6. 在 QMT 加载极速交易桥：加载 CFQUANT_TRADE_LOWLAT.py，用于下单、撤单和交易回调。
    7. 
                      
  7. 回到网页验证：进入“绑定”页面保存账号绑定，再刷新状态和资产持仓。
    8. 
                    

                  


                  

                    
多 QMT 部署

                    
如果一台机器上同时运行多个 QMT，每个 QMT 都要有自己能导入的 python\cfquant\ 目录、桥接入口脚本和 tx.py。每个 QMT 的 USER_BRIDGE_ID 必须不同。

                    
# 第二个 QMT 终端里的 CFQUANT.py 和 CFQUANT_TRADE_LOWLAT.py 保持一致
USER_BRIDGE_ID = "qmt2"

                    
改完后，在 Web 的“绑定”页面新增同名桥接端，例如 qmt2，再把对应资金账号绑定过去。

                  

                


                

                  

                    
QMT 图文部署指南

                    

                      推荐按本页顺序操作
                      先在 QMT 下载 Python 库，再创建两个 Python 策略，确认 tx.py 已放到 QMT 的 python 目录，最后启动本地交易端并回到网页验证。截图来自 cfquant\docs\部署指南.md，适合作为新用户部署时的主要教程。
                    

                    
复制文件时，目标目录一般是 QMT 安装目录下的 python 文件夹，例如 D:\兴业证券SMT-Q\python 或 D:\国金证券QMT交易端\python。文件放好后，再按下面的截图流程在 QMT 里完成策略创建和启动。

                  


                  

                    
一、下载 Python 库

                    
打开 QMT 后，进入“模型研究”或“模型交易”，点击“下载 Python 库”。这一步用于让 QMT 准备运行 Python 策略所需的环境。

                    
                  


                  

                    
二、创建两个 Python 策略

                    

                      建议创建两个独立策略
                      CFQUANT.py 负责普通查询桥，CFQUANT_TRADE_LOWLAT.py 负责极速交易桥。两个策略都需要放在 QMT 的 python 目录下，并在同一个 QMT 终端里加载。
                    

                    
在模型研究里点击“新建策略”，选择 Python 策略。按实际需要分别创建并加载 CFQUANT.py 和 CFQUANT_TRADE_LOWLAT.py。

                    
                  


                  

                    
三、放置 tx.py

                    

                      tx.py 必须在 QMT 的 python 根目录
                      桥接脚本启动时会导入 tx.py 来连接 LTtx 服务。如果 QMT 提示无法导入 tx，优先检查 tx.py 是否和 CFQUANT.py、CFQUANT_TRADE_LOWLAT.py 放在同一个 python 目录下。
                    

                    
把发布包 qmt_scripts 里的 tx.py 放到 QMT 安装目录的 python 文件夹，例如 D:\兴业证券SMT-Q\python\tx.py。

                    
                  


                  

                    
四、启动 cfquant 交易端

                    
回到外层 cfquant 发布目录，双击 start_cfquant.bat 启动 LTtx 和 Web 控制台。启动后浏览器会打开本地控制台。

                    
                  


                  

                    
五、网页端绑定账号

                    

                      启动后先绑定账号
                      打开 Web 控制台，一般地址是 http://127.0.0.1:8765/。进入“绑定”页面，把资金账号绑定到对应桥接端；单 QMT 通常使用 default。
                    

                    
保存绑定后，网页和外部接口就可以按账号自动找到对应的 QMT 桥接端。多 QMT 场景下，每个账号要绑定到实际登录它的 QMT 终端。

                    
                  


                  

                    
六、确认部署成功

                    
回到 Web 控制台，检查 LTtx、普通 QMT、极速交易端状态。保存资金账号和桥接端绑定后，刷新资产和持仓。如果能返回真实数据，说明部署已经打通。

桥接端配置

                    
桥接端 ID 是 Web Server 识别 QMT 终端的关键字段。普通桥和极速交易桥必须使用同一个 bridge_id，这样同一个账号的查询和交易才能落到同一套 QMT 环境里。

                    
如果只运行一个 QMT，保持默认桥接端即可。如果同时运行多个 QMT，例如一个负责实盘账号、一个负责测试账号，就给每个 QMT 分配不同桥接端 ID。

                    
# 示例：第二个 QMT 终端使用 qmt2
USER_BRIDGE_ID = "qmt2"

                  


                  

                    
推荐配置方式

                    

                      

                        
                          

                            场景
                            桥接端 ID
                            说明
                          

                        
                        
                          

                            单 QMT、单账号
                            default
                            最简单场景，不需要额外区分终端。
                          

                          

                            单 QMT、多账号
                            default
                            多个账号都绑定到同一桥接端，由账号字段区分。
                          

                          

                            多 QMT、多账号
                            qmt1、qmt2
                            每个 QMT 使用不同桥接端，每个账号绑定到实际登录它的终端。
                          

                        
                      

                    

                  

                


                

                  

                    
账号绑定

                    
账号绑定就是把 account_id 映射到 bridge_id。网页和外部程序发起请求时，只要携带资金账号，服务端就能找到对应 QMT 终端。这样外部程序不需要记住每个账号在哪个 QMT 里。

                    
    
                      
  1. 进入“绑定”页面，在“桥接端管理”里保存桥接端 ID 和显示名称。
    2. 
                      
  2. 在“账号绑定”里填写资金账号，选择桥接端并保存。
    3. 
                      
  3. 在“绑定状态”里刷新状态，确认普通和极速两个通道都在线。
    4. 
                      
  4. 点击验证按钮，检查资金和持仓是否来自预期账号。
    5. 
                    

                  


                  

                    
绑定规则

                    

                      一个账号只应绑定到一个当前可交易的桥接端
                      如果同一资金账号在多个 QMT 同时登录，回调和委托状态可能不容易判断来源。实盘使用时建议保持账号和桥接端关系清晰、稳定。
                    

                    
绑定保存后，概览页、交易页、接口页都会复用这套路由。后续切换账号时，只需要在顶部账号输入框输入目标账号，或从账号配对列表里选择。

                  

                


                

                  

                    
交易与回调

                    
交易页把下单、批量下单、撤单、持仓、委托和成交放在同一个工作区。查询通道和交易通道可以分别选择，通常查询走普通 QMT，交易走极速交易端。

                    

                      

                        
                          

                            操作
                            关键点
                          

                        
                        
                          

                            提交委托
                            填写代码、价格、数量，确认文本必须与页面提示完全一致，避免误操作。
                          

                          

                            批量买入
                            每行一笔委托，格式为 代码,价格,数量，确认文本使用 BATCH 数量。
                          

                          

                            撤单
                            实时委托表会标记可撤委托，可以选择后批量撤单，也可以双击可撤行触发撤单确认。
                          

                          

                            回调
                            委托、成交、下单错误、撤单错误会进入“回调”页，也可以通过 WebSocket 推给外部程序。
                          

                        
                      

                    

                  


                  

                    
实盘操作建议

                    
实盘前先用小数量验证账号、桥接端、价格和撤单链路。确认回调能返回后，再把外部程序接入自动交易流程。

                    
如果委托提交后页面没有立即变化，先看“回调”页和“日志”，再刷新“委托”。柜台状态、QMT 本地缓存和网络延迟都可能造成短暂不同步。

                  

                


                

                  

                    
外部接口

                    
“接口”页面提供在线调试。外部程序接入前，建议先在网页里选中同一个接口，填写参数并发送请求，确认返回结构和错误信息符合预期。

                    
启用 API Key 后，HTTP 请求需要携带 X-API-Key 请求头；WebSocket 可以通过查询参数 apikey 传入。

                    
curl -H "X-API-Key: your-api-key" ^
  "http://127.0.0.1:8765/api/account?account_id=2220009880§ions=asset,positions&force=1"

                  


                  

                    
常用接口

                    

                      

                        
                          

                            接口
                            用途
                          

                        
                        
                          

                            GET /api/account
                            查询资金、持仓、委托、成交，使用 sections 控制返回内容。
                          

                          

                            POST /api/order
                            提交单笔买入或卖出委托。
                          

                          

                            POST /api/orders/batch
                            一次提交多笔委托，适合外部策略批量执行。
                          

                          

                            POST /api/cancel
                            按委托编号撤单。
                          

                          

                            WS /ws/callbacks
                            实时接收委托、成交和错误回调。
                          

                        
                      

                    

                  

                


                

                  

                    
排查问题

                    
排查时先判断是哪一层断了：网页是否能访问，本地服务是否启动，LTtx 是否在线，QMT 脚本是否注册桥接端，账号绑定是否正确。

                    

                      

                        
                          

                            现象
                            优先检查
                            处理方式
                          

                        
                        
                          

                            网页打不开
                            Web Server 是否启动，端口是否被占用。
                            重新运行启动脚本，检查控制台日志和端口配置。
                          

                          

                            LTtx 离线
                            LTtx 进程是否运行，本地通信端口是否正常。
                            在“概览”或“状态”页点击启动 LTtx，再刷新状态。
                          

                          

                            普通在线、极速离线
                            CFQUANT_TRADE_LOWLAT.py 是否在 QMT 中加载。
                            重新加载极速交易桥，确认 bridge_id 与普通桥一致。
                          

                          

                            查不到账号数据
                            账号绑定是否保存到正确桥接端，QMT 是否登录该账号。
                            在“绑定”页重新保存账号绑定，并用验证功能检查资金和持仓。
                          

                          

                            下单失败
                            确认文本、交易权限、价格数量、交易通道。
                            查看日志和回调错误，先用小数量再次验证。
